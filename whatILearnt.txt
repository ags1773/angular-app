*************************************
**      Directives - Deep Dive     **
*************************************

Binding to directive properties:

@Input() defaultColor: string = 'transparent';
@Input() highlightColor: string = 'tomato';
.
.
<p appBetterHighlight [defaultColor]="'purple'" [highlightColor]="'green'">Better directive test para</p>

user can set default & highlight color from html using property binding.
Angular figures out that 'defaultColor' and 'highlightColor' are properties of the custom directive 'appBetterHighlight'.

---------------------------------------
Also note, this too works
@Input('appBetterHighlight') defaultColor: string = 'transparent';
.
.
<p [appBetterHighlight]="'purple'" [highlightColor]="'green'">Better directive test para</p>

here, we are giving "@Input() defaultColor" an alias that's same as the custom directive's name i.e. "appBetterHighlight"
So now, we can directly use property binding on the directive to pass defaultColor value i.e. [appBetterHighlight]="'purple'"
